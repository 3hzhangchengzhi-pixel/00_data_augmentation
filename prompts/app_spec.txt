================================================================================
项目规格说明书：二手车数据采集与价格预测 Pipeline
================================================================================

一、项目概述
================================================================================

本项目是一个完整的「数据采集 → 数据清洗 → 特征工程 → AI 估值模型」Pipeline，
目标是从日本二手车交易网站抓取车辆数据，经过清洗和特征提取后，训练机器学习模型
用于预测二手车价格。

参考实现：01_data_augmentation 文件夹（已实现爬虫采集部分）

二、技术栈
================================================================================

- 语言：Python 3.12+
- 包管理器：uv
- 爬虫：requests + BeautifulSoup4 + lxml
- 数据处理：pandas, numpy
- 可视化：matplotlib, japanize-matplotlib, japanmap, geopandas
- 数据库：MySQL (pymysql + sqlalchemy), PostgreSQL
- 机器学习：scikit-learn, xgboost, lightgbm
- 笔记本：JupyterLab
- BI 工具：Metabase（通过 Docker Compose）
- 测试：pytest
- 代码质量：ruff (lint + format)

四、核心模块详细说明
================================================================================

4.1 爬虫采集模块（websites/）
--------------------------------------------------------------------------------

参考 01_data_augmentation/src/data_augmentation/websites/ 中已实现的三个爬虫。
（注意，已經遷移到00_data_augmentation文件夾下）

目标网站：
  1. Carsensor (https://www.carsensor.net) — 日本最大二手车信息网站
  2. Mobilico (https://mobilico.jp) — 二手车交易平台
  3. AucSupport (https://www.aucsupport.com) — 拍卖支持平台

每个爬虫需提取的字段：
  - 车辆标题 (title)
  - 厂商/品牌 (maker)
  - 年式 (year)
  - 走行距离/里程 (mileage_km)
  - 车检到期日 (inspection_date)
  - 修复历史 (accident_history)
  - 保证/保修 (warranty)
  - 排气量 (displacement_cc)
  - 变速箱类型 (transmission)
  - 总额 (total_price)
  - 车辆本体价格 (base_price)
  - 地域 (region)
  - 颜色 (color)
  - 等级 (grade)

爬虫通用要求：
  - 请求间隔至少 3 秒（遵守网站礼仪）
  - 正确处理日语编码（UTF-8, Shift_JIS/CP932）
  - 异常处理：网络超时、页面结构变化、空数据
  - 输出格式：pandas DataFrame，保存为 pickle
  - 支持分页抓取，页数可配置
  - 日志记录抓取进度

4.2 数据清洗模块（cleaning/）
--------------------------------------------------------------------------------

cleaner.py — 统一清洗入口：
  - 加载各网站的原始 pickle 数据
  - 调用标准化和去重逻辑
  - 合并多源数据为统一 DataFrame
  - 输出清洗后的数据

normalize.py — 字段标准化：
  - 价格统一为日元（整数），去除「万円」等单位文字
  - 里程统一为公里（整数），处理「万km」等格式
  - 年式统一为西历年份（整数），处理和历转换
  - 车检日期解析为标准日期格式
  - 修复历史统一为布尔值（有/无）
  - 变速箱统一分类（AT/MT/CVT）
  - 地域名称标准化

dedup.py — 去重：
  - 基于车辆标题 + 价格 + 里程的组合去重
  - 跨数据源去重（同一辆车可能出现在多个网站）
  - 保留信息最完整的记录

4.3 特征工程模块（features/）
--------------------------------------------------------------------------------

numerical.py — 数值特征：
  - 车龄 = 当前年份 - 年式
  - 里程区间分桶（0-3万, 3-6万, 6-10万, 10万+）
  - 排气量分桶
  - 价格对数变换（用于模型训练）

categorical.py — 分类特征编码：
  - 地域 → 独热编码（One-Hot）或标签编码（Label）
  - 变速箱类型 → 独热编码
  - 颜色 → 标签编码
  - 厂商 → 标签编码

derived.py — 衍生特征：
  - 单位里程价格 = 价格 / 里程
  - 年均里程 = 里程 / 车龄
  - 折旧率估算
  - 是否有车检剩余（布尔）
  - 车检剩余月数

builder.py — 特征构建入口：
  - 组合以上所有特征
  - 处理缺失值（填充或删除策略）
  - 特征选择（去除高相关性冗余特征）
  - 输出训练就绪的特征矩阵 (X) 和目标变量 (y)

4.4 机器学习模型模块（models/）
--------------------------------------------------------------------------------

trainer.py — 模型训练：
  - 支持多种算法：
    * 线性回归 (LinearRegression) — 基线模型
    * 随机森林 (RandomForestRegressor)
    * XGBoost (XGBRegressor)
    * LightGBM (LGBMRegressor)
  - 训练/验证/测试集划分（70/15/15）
  - 交叉验证（5-Fold）
  - 训练过程日志

evaluator.py — 模型评估：
  - 评估指标：
    * RMSE（均方根误差）
    * MAE（平均绝对误差）
    * R²（决定系数）
    * MAPE（平均绝对百分比误差）
  - 多模型对比表格
  - 残差分析图
  - 特征重要性排序图
  - 预测值 vs 真实值散点图

predictor.py — 推理接口：
  - 加载已训练的模型文件
  - 输入车辆参数，输出预测价格
  - 输出预测置信区间

tuner.py — 超参数调优：
  - Grid Search 或 Random Search
  - 基于交叉验证的最优参数选择
  - 输出最优参数组合

4.5 数据存储模块（storage/）
--------------------------------------------------------------------------------

schema.py — 数据库表结构：
  - cars 表：车辆基本信息
  - prices 表：价格记录（历史追踪）
  - predictions 表：模型预测结果
  - scrape_logs 表：爬虫运行记录

database.py — 数据库操作：
  - SQLAlchemy ORM 连接
  - CRUD 操作封装
  - 支持 MySQL 和 PostgreSQL

export.py — 数据导出：
  - DataFrame → CSV
  - DataFrame → pickle
  - DataFrame → SQL (批量插入)
  - 模型 → joblib/pickle 序列化

六、测试要求
================================================================================

所有测试使用 pytest 框架执行。

测试类型：

  1. 单元测试 — 各函数独立测试
     - 清洗函数：输入脏数据 → 验证输出正确
     - 特征函数：输入 DataFrame → 验证特征计算正确
     - 存储函数：写入/读取 → 验证数据一致

  2. 集成测试 — 模块间协作
     - 爬虫 → 清洗 → 验证数据流通
     - 清洗 → 特征 → 模型 → 验证端到端 Pipeline

  3. 数据质量测试
     - 清洗后无空值（关键字段）
     - 价格范围合理（> 0 且 < 上限）
     - 年份范围合理
     - 里程非负
     - 无重复记录

测试运行命令：
  uv run pytest tests/ -v
